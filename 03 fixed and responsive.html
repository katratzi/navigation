Code Notes

We used the 'fixed' position for our unordered list in this case. So when scrolling it will always appear in the top left corner. You can position elements to any corner of the screen just based on the coordinates. Then so we don't get overlapping on a smaller screen, we use a @media query. In this case we have to override all the values to make sure looks correct.

Try scrolling and resizing.

// fixed unordered list ul { position: fixed; top: 1em; left: 20px; width: 200px; list-style-type: square; ... } // and when we get small...less than 800px // reset position to static (the default) // and set it like a normal navigation list @media only screen and (max-width: 800px) { ul { position: static; list-style-type: none; text-align: center; width: 100%; margin: auto; padding: 10px 10px 10px 10px; } li { display: inline; }
